Your CamCOPS server

  • CamCOPS will operate offline on your tablet, and you can view simple summaries of your tasks there.
  • You should set up a server to receive data from your CamCOPS tablet(s).
    • The simplest way (involving only free software) is to set up a Linux (e.g. Ubuntu) computer with an Apache web server and a MySQL database (plus the Python and Perl interpreters that come with most Linux distributions) – collectively known as a LAMP stack.
    • You must also obtain (or create for free) an HTTPS certificate, and configure your web server to do all its CamCOPS work via HTTPS only (not HTTP), so that your information is always encrypted in transit from the tablets to your server. CamCOPS will not use unencrypted links.
    • Then, there are a few CamCOPS scripts that you can add to your web server, so that it can receive data from your tablets, and offer a simple web front end so you can retrieve nice-looking PDFs of your tasks.
    • For more advanced analysis, you can then use the MySQL database directly.
  • Having set up your server, you should point your tablet(s) to it (see “Settings” → “Configure server settings”).
  • For details, see the CamCOPS website.

Locked and privileged modes

  • When you first install CamCOPS, you should enter access codes (see “Settings” → “Set access codes”).
  • At the top right of a screen is an unlocked padlock (indicating Unlocked Mode), a locked padlock (indicating Locked Mode), or two unlocked padlocks (Privileged Mode).
  • Having selected a patient, you can lock your tablet by touching the padlock. You can then safely give it to that patient, and they will not be able to view other patients’ details.
  • To unlock the tablet, touch the padlock and enter the code.
  • Privileged mode (see “Settings”) is required to configure server settings and the like.
